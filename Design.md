# Design — Visual Identity

This project has no GUI — its "interface" is PR comments, PR descriptions, terminal
output, and the README. Those surfaces still deserve deliberate design.

## 1. Brand

- **Name**: `dochealer` (package), **Self-Healing Docs** (display name)
- **Tagline**: "Docs that fix themselves when your code changes."
- **Action marketplace branding** (`action.yml`): icon `book-open`, color `green`
  (green = healing/health; book = docs).

## 2. PR Comment Design

Comments are the product's face. Rules:

- **Lead with the verdict, not the process.** First line is the result summary.
- **Emoji as status glyphs, sparingly**: ✅ verified · 🩹 auto-fixed · ⚠️ needs review ·
  ⏭️ skipped. Never decorative emoji.
- **Everything links**: section names link to file+line, fix PRs link directly.
- **Collapsible detail**: diagnoses inside `<details>` blocks so the comment stays short.
- Hidden marker `<!-- dochealer-summary -->` for comment upserts (edit, don't spam).

Template:

```markdown
### 🩺 Doc Check Results
✅ 3 sections verified accurate · 🩹 1 auto-fixed → #42 · ⚠️ 2 flagged for review

<details><summary>⚠️ Sections needing review</summary>

| Section | Why it's suspect |
|---|---|
| [Configuration › Timeouts](docs/config.md#L88) | `retry_delay` default changed 5→30 s |
</details>

<sub>dochealer v1 · 4 code changes analyzed · 2 LLM verifications · ~$0.03</sub>
```

## 3. Terminal / Log Output

- Structured stage headers: `[index]`, `[detect]`, `[repair]`, `[report]`.
- One line per decision: `[detect] src/app.py::get_user — signature change → 2 suspect sections`.
- Summary block at end mirroring the PR comment counts.
- No color codes in CI logs (GitHub renders them poorly); rely on prefixes.

## 4. README / Docs Typography

- Heading case: sentence case everywhere ("Change detection", not "Change Detection").
- Code voice: all identifiers, paths, flags in backticks.
- Diagrams: ASCII (renders in every context, matches Architecture.md).
- Badges row: CI status, marketplace version, license — max 4 badges.
- Demo GIF placed within the first screenful, right after the one-paragraph pitch.

## 5. Fix-PR Description Design

```markdown
## 🩹 Doc fixes for #<n>

`docs/config.md` › **Configuration › Timeouts**
> `retry_delay` default updated 5 → 30 s (changed in `src/config.py::Settings`)

_Generated by dochealer · validated · confidence 0.93_
Review checklist: [ ] accurate  [ ] style preserved  [ ] nothing over-deleted
```

- One block per fixed section; quote the *reason*, not the whole diff (the diff tab shows it).
- Always include the human review checklist — signals humility, invites verification.

## 6. Naming Conventions (code-level design)

- Branches: `dochealer/fix-pr-<n>`; label: `dochealer`.
- Index dir: `.dochealer/` (dot-prefixed, tool-owned, like `.github/`).
- Config env vars: `DOCHEALER_*` prefix.
